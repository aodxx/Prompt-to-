## ขั้นตอนที่ 1 — ตั้งค่า Google Apps Script (หลังบ้าน)

1. เปิด Google Sheet ตาม ID ด้านบน
2. เมนู **ส่วนขยาย (Extensions) → Apps Script**
3. ลบโค้ดเดิมทั้งหมด แล้ววางไฟล์ `Code.gs` ที่แนบมาแทน
4. กด **Deploy → New deployment**
   - ไอคอนเฟือง → เลือกประเภท **Web app**
   - Execute as: **Me**
   - Who has access: **Anyone**
   - กด **Deploy** แล้วอนุญาตสิทธิ์ (Authorize access) ตามที่ระบบขอ (สิทธิ์เข้าถึง Sheet และ Drive)
5. คัดลอก **Web app URL** ที่ได้ (รูปแบบ `https://script.google.com/macros/s/xxxxx/exec`)

> ทุกครั้งที่แก้ไขโค้ด `Code.gs` ต้องกด **Deploy → Manage deployments → แก้ไข (ดินสอ) → Version: New version → Deploy** ใหม่ URL เดิมจะใช้งานต่อได้

Sheet จะถูกสร้างชีตชื่อ `Prompts` อัตโนมัติพร้อมหัวตารางในการเรียกใช้ครั้งแรก ไม่ต้องสร้างเองล่วงหน้า

### ตั้งค่า Gemini API Key (สำหรับฟีเจอร์ AI ขัดเกลา Prompt)

1. ไปที่ [Google AI Studio](https://aistudio.google.com/apikey) เพื่อสร้าง API Key ฟรี (ล็อกอินด้วย Google account เดียวกับ Sheet ก็ได้)
2. กลับไปที่ Apps Script Editor → ไอคอนเฟือง **Project Settings** ทางซ้าย
3. เลื่อนลงมาส่วน **Script Properties** → **Add script property**
   - Property: `GEMINI_API_KEY`
   - Value: (วาง API Key ที่คัดลอกมา)
4. กด Save — ไม่ต้องแก้โค้ดใดๆ เพิ่ม ระบบจะดึงค่านี้มาใช้อัตโนมัติ

> เก็บ Key ไว้ใน Script Properties แทนการฝังในโค้ดตรงๆ เพื่อไม่ให้ Key หลุดไปกับไฟล์ที่ push ขึ้น GitHub (repo หน้าบ้านเป็น public)

ค่าเริ่มต้นใช้โมเดล `gemini-2.5-flash` (เสถียร เร็ว ราคาประหยัด) หากต้องการโมเดลรุ่นใหม่กว่า แก้ตัวแปร `GEMINI_MODEL` ใน `Code.gs` เป็น `gemini-3.5-flash` ได้ (ต้อง Deploy เวอร์ชันใหม่ทุกครั้งที่แก้)

## ขั้นตอนที่ 2 — ใส่ Web App URL ในหน้าบ้าน

เปิดไฟล์ `index.html` หาบรรทัดนี้ (อยู่ในส่วน `<script>` ท้ายไฟล์):

```js
const CONFIG = {
  API_URL: 'https://script.google.com/macros/s/PUT_YOUR_DEPLOYMENT_ID_HERE/exec'
};
```

แทนที่ด้วย Web App URL ที่คัดลอกมาจากขั้นตอนที่ 1

repo ที่ใช้คือ **`Prompt-to-`** (ชื่อมีขีดกลางต่อท้าย ตามที่ตั้งไว้) ไฟล์ทุกไฟล์ในโปรเจ็กต์นี้อ้างอิง path แบบ absolute เป็น `/Prompt-to-/...` ไว้แล้ว (ตรงกับพฤติกรรมของ GitHub Pages ที่ไม่ได้อยู่ root domain)

## โครงสร้างข้อมูลใน Sheet (`Prompts`)

| คอลัมน์ | ความหมาย |
|---|---|
| ID | รหัสอ้างอิง (สร้างอัตโนมัติ) |
| Name | ชื่อ Template |
| Category | หมวดหมู่ |
| Tags | แท็ก คั่นด้วยจุลภาค |
| Content | เนื้อหา Prompt ใช้ `{{ชื่อตัวแปร}}` สำหรับส่วนที่ต้องการให้กรอกทีหลัง |
| Description | คำอธิบายเพิ่มเติม (ยังไม่แสดงผลใน UI เวอร์ชันนี้ เผื่อขยายภายหลัง) |
| ImageURL | ลิงก์รูปตัวอย่าง (สำหรับ Prompt ประเภทสร้างภาพ) |
| DemoURL | ลิงก์ผลงานที่เคยได้จาก Prompt นี้ — เช่น แอป Interactive หน้าเดียว หรือเกม 3D ที่ deploy ไว้แล้ว บนหน้าแอปจะมีปุ่ม "↗ ดูผลงานที่เคยได้" ให้กดเปิดตรงจากรายการ |
| CreatedAt / UpdatedAt | เวลาสร้าง/แก้ไขล่าสุด |

**ตัวอย่างการเขียน Content:**
```
ช่วยเขียนบทความเรื่อง {{หัวข้อ}} ความยาว {{จำนวนคำ}} คำ โทนเสียงแบบ {{โทนเสียง}}
```
แอปจะสร้างช่องกรอก 3 ช่องให้อัตโนมัติในโหมดฟอร์ม

> **หมายเหตุการอัปเดต:** ถ้า Sheet ของคุณถูกสร้างไว้ก่อนที่จะมีคอลัมน์ `DemoURL` ไม่ต้องแก้อะไรเอง — โค้ดฝั่งหลังบ้านเวอร์ชันนี้จะตรวจสอบและเพิ่มคอลัมน์ที่ขาดต่อท้ายให้อัตโนมัติในการเรียกครั้งแรก โดยไม่กระทบข้อมูลเดิมที่มีอยู่

## ฟีเจอร์ที่มีอยู่แล้วในเวอร์ชันนี้

- ค้นหาแบบ Real-time + กรองตามหมวดหมู่
- ตัวแปรอัตโนมัติจาก `{{...}}` พร้อมโหมดฟอร์ม/โหมดข้อความอิสระ
- **Gemini Co-Pilot:** พิมพ์คำสั่งเพิ่มเติม (เช่น "เน้นโทนจริงจังขึ้น") แล้วกด "ให้ Gemini ปรับแต่ง" ระบบจะส่งชื่อสไตล์ + หมวดหมู่ + เนื้อหาที่กรอกในฟอร์มแล้ว + คำสั่งเพิ่มเติม ไปให้ Gemini ผ่าน Apps Script (`UrlFetchApp`) แล้วนำผลลัพธ์มาแสดงในกล่องผลลัพธ์
- **ผลลัพธ์แก้ไขได้โดยตรง (Editable Preview):** กล่องผลลัพธ์เป็น textarea แก้ไขมือสกรีนสุดท้ายได้ก่อนคัดลอก เมื่อแก้ค่าตัวแปรในฟอร์มใหม่ จะมีปุ่ม "🔄 ซิงค์ค่าจากฟอร์มอีกครั้ง" ให้กดถ้าต้องการทิ้งผลลัพธ์เดิมแล้วดึงค่าฟอร์มมาแทน (ป้องกันฟอร์มไปทับผลลัพธ์ที่ Gemini เพิ่งปรับให้โดยไม่ตั้งใจ)
- **บันทึกเป็นสไตล์ใหม่ (Save as New Style):** ถ้าผลลัพธ์จาก Gemini ถูกใจ กดปุ่มนี้เพื่อตั้งเป็น Template ใหม่ (ตั้งชื่ออัตโนมัติว่า "ชื่อเดิม (Gemini)" แก้ไขได้) แล้วกด "บันทึก" อีกครั้งเพื่อเซฟลง Google Sheet เป็นแถวใหม่แยกจากต้นฉบับ
- ปุ่ม Copy to Clipboard พร้อม Toast แจ้งเตือน
- แนบรูปตัวอย่างสำหรับ Prompt สร้างภาพ (อัปโหลดขึ้น Google Drive ผ่าน hidden-iframe POST เพื่อเลี่ยงปัญหา CORS ของ Apps Script)
- ติดตั้งเป็นแอป (PWA) ได้ทั้งมือถือและเดสก์ท็อป

## จุดที่ยังออกแบบไว้แบบพื้นฐาน รอส่วนขยายภายหลัง

- ระบบ Favorite / จำนวนการใช้งาน (Usage Count) — ยังไม่ได้ทำ เพราะรอบแรกเลือกโครงสร้างฟิลด์แบบพื้นฐาน (ชื่อ/หมวดหมู่/เนื้อหา) แต่เผื่อคอลัมน์ `Description` และ `ImageURL` ไว้ให้แล้วจากสเปกที่ส่งมาเพิ่ม
- การทำงานแบบออฟไลน์ตอนนี้แคชได้เฉพาะหน้าเปล่า (App Shell) ข้อมูล Prompt และการเรียก Gemini ยังต้องใช้อินเทอร์เน็ตทุกครั้ง
- เมื่อกด "บันทึกเป็นสไตล์ใหม่" ระบบจะเซฟข้อความที่ Gemini ปรับแล้ว (ซึ่งมักไม่มี `{{ตัวแปร}}` หลงเหลือ เพราะถูกแทนที่ค่าไปแล้ว) หากต้องการให้สไตล์ใหม่นี้มีตัวแปรสำหรับใช้ซ้ำได้อีก ให้แก้ไขใส่ `{{...}}` กลับเข้าไปในช่องเนื้อหาก่อนกดบันทึก
